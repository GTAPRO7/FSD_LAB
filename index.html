System Architecture

The Campus Navigation System is engineered to optimize student commutes and eliminate navigational friction across complex university grounds.

Database Models

We leverage MongoDB's powerful spatial querying capabilities. Building locations and pathways are stored as GeoJSON objects, utilizing 2dsphere indexing to allow for rapid proximity calculations. 

// Sample Node Schematic
{
  "name": "Science Building",
  "location": {
    "type": "Point",

  }
}

API Interactivity

The Express server exposes a RESTful API. When the React client submits a start and end destination, the backend executes our custom pathfinding middleware, returning an ordered array of coordinate waypoints to render the polyline on the frontend map.
